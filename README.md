# AICA SQL Assessment (PostgreSQL)
This project contains my solutions to a SQL assessment based on a Nigerian e-commerce dataset. The analysis focuses on extracting business insights using PostgreSQL.

## Objectives
- Analyze customer distribution and behavior
- Identify high-value and dormant customers
- Evaluate product performance and revenue contribution
- Build sales performance metrics for employees
- Detect unusual or suspicious transactions

## Tools Used
- PostgreSQL
- pgAdmin
- SQL (joins, aggregations, window functions, CTEs)

## Project Structure
- `Oluwabukunmi_SQL_Assessment.sql` — all 10 queries with approach comments
- `SQL_Assessment_Dataset.xlsx` — the raw dataset (five sheets, one per table)
- `SQL_Assessment_AICA.pdf` — the original question sheet

## Key Highlights
- Used window functions to implement the 80/20 revenue rule (Pareto analysis)
- Built customer segmentation logic (one-time vs repeat buyers)
- Identified dormant high-value customers for potential re-engagement
- Designed a sales performance scorecard for internal evaluation
- Detected anomalous orders using customer-level benchmarks
